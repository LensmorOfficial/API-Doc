# Error conventions

External API errors use real HTTP status codes.

## Error body
```json
{
  "code": 401,
  "message": "Invalid API key",
  "errorKey": "INVALID_API_KEY",
  "traceId": "2be6d7f4-3b2d-4d11-8e2d-7a2e4c3e9a10"
}
```

## Notes
- External responses include `traceId`.
- External error bodies do not expose internal `data` or `details` fields.
- Common statuses across these endpoints include `400`, `401`, `404`, and `409`, depending on route behavior.
