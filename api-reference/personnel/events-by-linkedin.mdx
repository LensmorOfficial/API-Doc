# Personnel related events by LinkedIn

Resolve a person from their LinkedIn profile URL and return their related events.

Use this endpoint when you have a LinkedIn profile URL instead of a `personnel_id` and need both the matched personnel profile and their associated events.

## Endpoint
`GET /external/personnel/events/by-linkedin`

## Authentication
See [Authentication](/authentication)

## Success status code
`200 OK`

## Query parameters
| Name | Required | Type | Notes |
| --- | --- | --- | --- |
| `linkedin_url` | Yes | string | Provide a valid LinkedIn profile URL. Query strings, fragments, and trailing slashes are ignored. |
| `page` | No | integer | Defaults to `1`. |
| `pageSize` | No | integer | Defaults to `20`; maximum `100`. |

## Error responses
- `400 Bad Request`
- `401 Unauthorized`
- `404 Not Found`

## Notes
- `personnel` reuses the same public profile shape as [Personnel profile details](/api-reference/personnel/profile), including `email` and `contactUnlockStatus`.
- `events` reuses the same paginated event-item shape as [Personnel related events](/api-reference/personnel/events).
- If the matched person has no related events, `events` returns an empty paginated success response rather than `404 Not Found`.
