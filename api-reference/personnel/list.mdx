# Personnel list

List personnel records for a specific event.

Use this endpoint when you want a paginated people directory for one event, with optional exhibitor, department, level, and search filters.
The response now includes a `semantics` object that explains preview versus full event access.

## Endpoint

`GET /external/personnel/list`

## Authentication

See [Authentication](/authentication)

## Success status code

`200 OK`

## Query parameters

| Name | Required | Type | Notes |
| --- | --- | --- | --- |
| `event_id` | Yes | string | Event identifier returned by event endpoints. |
| `exhibitor_id` | No | string | Filter by a specific exhibitor. |
| `department` | No | string | Department filter. |
| `level` | No | string | Seniority-level filter. |
| `search_query` | No | string | Name or job-title search. |
| `page` | No | integer | Defaults to `1`. |
| `pageSize` | No | integer | Defaults to `20`; maximum `100`. |


## Error responses

- `400 Bad Request`
- `401 Unauthorized`
- `404 Not Found`

## Notes
- Each item can include user-scoped `email` and `contactUnlockStatus`.
- `semantics.accessMode` indicates whether this query is still in preview mode or already has full event access.
- In preview mode, `semantics.previewLimit` is currently `50`, and `semantics.unlock` / `semantics.guidance` explain whether event unlock is required to see more results.
- If `semantics.pageState.accessible` is `false`, the requested page can be empty even when more matching results exist behind the preview boundary.
- Pagination behavior follows the shared [Pagination conventions](/concepts/pagination).
- `title` returns the role or title shown for that event record.
- `search_query` matches names and role or title text.
