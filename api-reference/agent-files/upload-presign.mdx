# Agent files upload presign

Create a one-time direct-upload URL for a generated file.

Use this endpoint to start the file upload flow when you need a presigned PUT URL and a `fileId` that can later be confirmed.

## Endpoint
`GET /external/agent-files/upload-presign`

## Authentication
See [Authentication](/authentication)

## Success status code
`200 OK`

## Query parameters
| Name | Required | Type | Notes |
| --- | --- | --- | --- |
| `name` | Yes | string | Original file name to associate with the upload record. |
| `contentType` | Yes | string | MIME type to use for the presigned PUT upload. |
| `instanceId` | No | string | Optional agent instance identifier for file history scoping. |

## Response body
| Field | Type | Notes |
| --- | --- | --- |
| `fileId` | string | File identifier used by [Agent files confirm upload](/api-reference/agent-files/confirm-upload). |
| `putUrl` | string | Presigned S3 PUT URL for direct binary upload. |
| `s3Key` | string | Storage object key assigned to the file. |

## Response example
```json
{
  "fileId": "1024",
  "putUrl": "https://storage.example.com/lensmor-test/agent-files/42/1024_report.csv?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Expires=600",
  "s3Key": "agent-files/42/1024_report.csv"
}
```

## Error responses
- `401 Unauthorized`

## Notes
- This is step 1 of the upload flow: request the presigned URL here, upload the file bytes to `putUrl`, then call [Agent files confirm upload](/api-reference/agent-files/confirm-upload).
- The current presigned PUT URL expiry is 10 minutes.
- If `instanceId` is omitted, the service may associate the file with your most recent active agent instance when one is available.
- If `name` or `contentType` is missing, the current implementation returns `200 OK` with an `error` field instead of a validation error response.
